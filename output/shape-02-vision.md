# Visão de Produto — Shape Stage 2

## Visão

Para líderes de time que precisam reportar progresso para gestores que não acompanham o trabalho diário, o Kanvas Weekly é uma atualização semanal estruturada que coloca a narrativa do time em primeiro lugar — com os dados automáticos do projeto como contexto, não como protagonistas.

## É / Não é

| É | Não é |
|---|-------|
| Um espaço para o líder de time contar o que aconteceu antes dos dados chegarem ao gestor | Uma ferramenta de avaliação de performance ou prestação de contas individual |
| Um complemento à automação de dados já existente do Kanvas | Um substituto para o tracking de tasks, boards ou sprints |
| Uma atualização semanal com estrutura fixa (narrativa + dados) | Uma ferramenta de comunicação geral ou mensageiro de equipe |
| A solução para o ciclo de abandono gerado pela má representação do trabalho | Uma reescrita completa da feature de Relatórios Automáticos |
| Parte do fluxo existente do Kanvas, não um produto separado | Um sistema de templates customizáveis por time |

## Faz / Não faz

| Faz | Não faz |
|-----|---------|
| Abre o preenchimento semanal com um campo de narrativa em destaque — o que aconteceu, o que ficou para trás, o que precisa de atenção — antes de exibir os dados automáticos | Processa o texto com IA para gerar sugestões ou resumos automáticos (v1) |
| Monta o relatório com estrutura definida: [narrativa do time] → [dados automáticos do projeto] | Permite campos customizáveis por time ou por empresa (v1) |
| Envia notificação ao gestor quando o relatório estiver disponível para leitura | Cria loop de feedback estruturado do gestor para o líder (v1 — adiado) |
| Mantém histórico de relatórios anteriores acessível ao líder para referência no preenchimento | Integra com ferramentas externas (Slack, email) — permanece dentro do Kanvas |
| Funciona para o fluxo existente — líder preenche, gestor lê — sem mudar permissões ou estrutura de time | Requer migração de dados ou reconfiguração pelos administradores |

## Usuário-alvo

**Primário:** líder de time (tech lead, coordenador, manager) em empresa de 50–300 funcionários que tem contexto sobre o trabalho da semana e precisa transmitir esse contexto para um gestor que não acompanha o dia a dia. Hoje esse usuário ou constrói workarounds fora do Kanvas ou preenche o mínimo por obrigação — e tem NPS -8 com a feature.

**Secundário:** gestor (COO, VP, Head de Área) que precisa de visibilidade de múltiplos times sem reunião de status semanal. Tem NPS +34 com a feature mas perde o valor quando o conteúdo que chega é raso ou inexistente.

## Métricas de sucesso

1. **`report_configured` volta ao baseline de Janeiro** (de 5.230 para 8.000+ eventos/mês) — medido em analytics mensalmente, avaliado 90 dias pós-lançamento
2. **Taxa de relatórios enviados no prazo sobe de 52% para 70%+** — medido em analytics semanalmente, meta em 60 dias pós-lançamento
3. **Churn mid-market (50–300 func.) cai abaixo de 4%** (de 6,8%) — medido em CRM/billing, avaliado no quarter seguinte ao lançamento
