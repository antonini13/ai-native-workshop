01

O Problema

Líderes de time não conseguem comunicar o contexto real do seu trabalho semanal porque o Kanvas posiciona dados automáticos de projeto como conteúdo principal do relatório e narrativa do time como detalhe secundário — fazendo o relatório representar mal o trabalho, desincentivando o preenchimento, e quebrando o ciclo de visibilidade que o produto prometia.

Top 3 Evidências

1

report_exported_csv cresceu +214,7% em 6 meses (340 → 1.070 eventos)

Prova que o problema não é falta de motivação: usuários fazem o trabalho, só que fora do produto. Export CSV é preditor de churn — contas que cancelaram têm 4,2× mais eventos de export que as ativas.

2

NPS líderes de time: -8 vs gestores: +34 · Queda assimétrica: -31% líderes vs -5,7% gestores

A insatisfação está cirurgicamente localizada no produtor do conteúdo. O gestor ainda tenta consumir, mas o líder já abandonou. Colapso pelo lado da oferta, não da demanda.

3

5 sistemas paralelos independentes criados por usuários: email manual, WhatsApp, Slack, Notion, planilha

Convergência independente para a mesma estrutura — narrativa livre + dados como apoio — é o briefing implícito do que o Kanvas precisaria ser.

02

O Insight

Os líderes de time não são preguiçosos nem relutantes. São motivados o suficiente para construir sistemas paralelos trabalhosos — WhatsApp, Slack, planilhas, emails semanais. A disposição de comunicar existe. O que falta é o formato certo dentro do produto.

O design priorizou o gestor (consumidor), mas quem decide se o produto tem conteúdo é o líder de time (produtor). A simplificação do formulário e a campanha de reengajamento falharam porque não tocaram nessa assimetria estrutural.

Pergunta de design

"Como poderíamos permitir que o líder de time conte a história do que aconteceu antes dos dados automáticos chegarem ao gestor, sem aumentar a carga de trabalho de preenchimento?"

03

A Visão de Produto

Para líderes de time que precisam reportar progresso para gestores que não acompanham o trabalho diário, o Kanvas Weekly é uma atualização semanal estruturada que coloca a narrativa do time em primeiro lugar — com os dados automáticos do projeto como contexto, não como protagonistas.

É

  • ✓ Espaço para narrativa do time antes dos dados chegarem
  • ✓ Complemento à automação de dados existente
  • ✓ Atualização semanal com estrutura fixa
  • ✓ Parte do fluxo existente do Kanvas

Não é

  • ✕ Avaliação de performance individual
  • ✕ Substituto para boards ou sprints
  • ✕ Mensageiro de equipe ou Slack interno
  • ✕ Reescrita completa da feature atual
